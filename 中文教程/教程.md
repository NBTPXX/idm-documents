# IDM 传感器安装与配置教程

## 安装前提

- Klipper 运行在 Python 3.6 或以上版本
- 使用本模块需要具备一定的 Klipper 使用经验和配置修改能力
- 为保证精度，安装时应尽量让传感器线圈板的顶面低于加热块的底面

> **重要提醒：教程里没有提到的操作不要做（尤其是 G28），请严格按照教程步骤执行。**

---

## 一、安装

在用户目录下执行以下 git 命令下载配套脚本：

```bash
git clone https://gitee.com/NBTP/IDM.git
```

如果你不确定 pip 源是否可用（或完全不了解 pip），建议先将 pip 设置为国内镜像源：

```bash
~/klippy-env/bin/pip config set global.index-url https://mirrors.aliyun.com/pypi/simple/
```

然后执行安装脚本：

```bash
IDM/install.sh
```

## 二、配置示范（新版本，使用 [scanner]）

将以下配置添加到 Klipper 的 `printer.cfg` 中：

```ini
[mcu idm]
serial:
#canbus_uuid:
# IDM 设备的串口号，通常格式为 /dev/serial/by-id/usb-idm_idm_...

[scanner]
mcu: idm
# IDM 的 MCU 名称
speed: 40.
# Z 探测下降速度
lift_speed: 5.
# Z 探测抬升速度
backlash_comp: 0.5
# 回差补偿距离，用于在测量传感器响应前消除 Z 回差
x_offset: 0.
# IDM 相对于喷嘴的 X 方向偏移
y_offset: 21.1
# IDM 相对于喷嘴的 Y 方向偏移
trigger_distance: 2.
# IDM 归零触发距离
trigger_dive_threshold: 1.5
# 量程探测与下潜探测的阈值，超过 trigger_distance + trigger_dive_threshold 时将使用下潜模式
trigger_hysteresis: 0.006
# 触发解除的回滞量，以触发阈值的百分比表示
cal_nozzle_z: 0.1
# 手动 Z 偏移校准完成后预期的喷嘴偏移
cal_floor: 0.1
# 传感器响应测量的 Z 轴下限
cal_ceil: 5.
# 传感器响应测量的 Z 轴上限
cal_speed: 1.0
# 测量响应曲线时的速度
cal_move_speed: 10.
# 移动到响应曲线测量位置时的速度
default_model_name: default
# 默认加载的 IDM 模型名称
mesh_main_direction: x
# 网床扫描时的主要移动方向
#mesh_overscan: -1
# 网床扫描线末端改变方向时的过冲距离。省略此项将根据线间距和可用行程自动计算
mesh_cluster_size: 1
# 网床扫描点聚类的半径
mesh_runs: 1
# 网床扫描的遍历次数
calibration_method: touch
# 校准方法设置。如果不使用 touch 模式，可以设为 scan 以兼容旧版本命令
sensor: idm
# 传感器名称，可选 idm 或 cartographer
scanner_touch_max_temp: 180
# 戳床时喷嘴降到该温度
scanner_touch_speed: 5
# 使用 touch 模式时的下降速度，不建议设置过高
scanner_touch_accel: 100
# 使用 touch 模式时的加速度
```

## 三、重要配置说明

### 3.1 修改串口号

配置中的 `x_offset` 和 `y_offset` 需要根据你的实际安装位置调整。确保校准过程中喷头会将线圈移动到原先喷嘴所在的 XY 位置。

查询 IDM 的串口号：

```bash
ls /dev/serial/by-id/*
```

将查询到的结果填入 `serial:`。

### 3.2 CAN 版本配置

CAN 版本使用 `canbus_uuid` 代替 `serial`。使用以下命令搜索 CAN 的 UUID：

**CanBoot（旧版）：**

```bash
~/klippy-env/bin/python ~/klipper/lib/canboot/flash_can.py -q
```

**Katapult（新版）：**

```bash
~/klippy-env/bin/python ~/klipper/lib/katapult/flashtool.py -i can0 -q
```

**注意：填入 UUID 后请删掉 `serial:` 这一行。**

## 四、必要配置

### 4.1 启用 force_move

在配置中加入以下内容（必须）：

```ini
[force_move]
enable_force_move: true
```

### 4.2 移除旧探针配置

1. 删除 `[probe]` 模块
2. 如果使用过 Klicky，请移除相关脚本的引用
3. 将 Z 限位（`stepper_z` 中的 `endstop_pin:`）修改为 `probe:z_virtual_endstop`

### 4.3 设置 safe_z_home

```ini
[safe_z_home]
home_xy_position: <你的X轴中心坐标>,<你的Y轴中心坐标>
z_hop: 10
```

如果已经配置过 `safe_z_home` 或 `homing_override`，可以跳过此步骤。

### 4.4 设置 bed_mesh

务必配置 `[bed_mesh]`，否则会报错。

## 五、校准流程

### 5.1 初始化 Z 位置

1. 重启 Klipper 后，归零 X 和 Y（执行 `G28 X Y`，**不要归零 Z**）
2. 将打印头移动到热床正中央
3. 执行 `SET_KINEMATIC_POSITION z=80`
4. 控制 Z 方向移动，将喷嘴贴到平台上（可垫 A4 纸确保间隙合适）
5. 执行 `SET_KINEMATIC_POSITION z=0`（注意：与上一条命令不同）

### 5.2 执行校准

**普通模式：**

```gcode
IDM_CALIBRATE
```

**Touch 模式**（设置了 `calibration_method: touch`）：

```gcode
IDM_TOUCH METHOD=MANUAL
```

在弹出的偏移控制框中，将打印头降到喷嘴贴热床，然后点击 -0.1 偏移后确认，校准将自动进行。

### 5.3 常见问题

如果校准后重启无法归零，报错 `no model`，说明配置文件的自动生成配置格式有误，请检查并修正格式。

## 六、大功率交流热床（500W 以上）

如果你的热床功率超过 500W（交流热床），建议配置以下宏来避免热床对扫床产生干扰：

```ini
[gcode_macro BED_MESH_CALIBRATE]
rename_existing: _BED_MESH_CALIBRATE
gcode:
    {% set TARGET_TEMP = printer.heater_bed.target %}
    M140 S0
    _BED_MESH_CALIBRATE {rawparams}
    M140 S{TARGET_TEMP}
```

## 七、多 Z 轴机器配置

### VORON 2.4 等 4Z 机器（QGL）

```ini
[gcode_macro QUAD_GANTRY_LEVEL]
rename_existing: _QUAD_GANTRY_LEVEL
gcode:
    SAVE_GCODE_STATE NAME=STATE_QGL
    BED_MESH_CLEAR
    {% if not printer.quad_gantry_level.applied %}
      _QUAD_GANTRY_LEVEL horizontal_move_z=10 retry_tolerance=1
    {% endif %}
    _QUAD_GANTRY_LEVEL horizontal_move_z=2
    RESTORE_GCODE_STATE NAME=STATE_QGL
```

### VORON Trident 等 3Z 机器（Z_TILT）

```ini
[gcode_macro Z_TILT_ADJUST]
rename_existing: _Z_TILT_ADJUST
gcode:
    SAVE_GCODE_STATE NAME=STATE_Z_TILT
    BED_MESH_CLEAR
    {% if not printer.z_tilt.applied %}
      _Z_TILT_ADJUST horizontal_move_z=10 retry_tolerance=1
    {% endif %}
    _Z_TILT_ADJUST horizontal_move_z=2
    RESTORE_GCODE_STATE NAME=STATE_Z_TILT
```

## 八、Moonraker 自动更新配置

建议在 `moonraker.conf` 中添加以下配置，便于自动更新 IDM 脚本：

```ini
[update_manager idm]
type: git_repo
channel: dev
path: ~/IDM
origin: https://gitee.com/NBTP/IDM.git
env: ~/klippy-env/bin/python
requirements: requirements.txt
install_script: install.sh
is_system_service: False
managed_services: klipper
info_tags:
  desc=idm
```

## 九、加速度计配置

> 加速度计的配置务必放置在 IDM 配置的后面。

### lis2dw 版本（方形芯片）

```ini
[lis2dw]
cs_pin: idm:PA3
spi_bus: spi1

[resonance_tester]
accel_chip: lis2dw
probe_points:
    125, 125, 20  # 设置为进行共振测量时喷头所处坐标
```

### adxl345 版本（长方形芯片）

```ini
[adxl345]
cs_pin: idm:PA3
spi_bus: spi1

[resonance_tester]
accel_chip: adxl345
probe_points:
    125, 125, 20  # 设置为进行共振测量时喷头所处坐标
```

配置完成后使用 `SHAPER_CALIBRATE` 进行共振测量。

## 十、Z 偏移调整

准备工作结束后，调整 Z 偏移再开始打印。Z 偏移保存在 `model_offset` 变量中。

> **调整 Z 偏移前务必关闭网床、完成机械调平，调平后需再归零一次。**

推荐使用 [axis_twist_compensation](https://www.klipper3d.org/Config_Reference.html#axis_twist_compensation) 来确保网床效果。

## 十一、管理多个校准结果（不同 PEI 板切换）

| 操作 | 命令 |
|------|------|
| 保存当前校准 | `IDM_MODEL_SAVE NAME=<名称>` |
| 载入已保存的校准 | `IDM_MODEL_SELECT NAME=<名称>` |
| 列出所有校准 | `IDM_MODEL_LIST` |
| 删除校准 | `IDM_MODEL_REMOVE NAME=<名称>` |

使用场景：更换不同 PEI 板时，可保存各自的校准数据，切换时直接载入，避免反复校准。
